Using AI to Perform Code Changes Directly on Source Files (Aider)

Jul 21, 2024

I have mixed feelings on using AI-powered tools for generating source code.

Tools like GitHub Copilot and Codeium, which provide auto-completion in editors and IDEs, can be helpful but also distracting when they don’t align with your train of thought.

The best results I’ve had were from chatting with ChatGPT for specific parts of my code, like rapid prototyping some class or functions and generating unit tests. However, constantly copying and pasting from the chatbot interface to my local editor can be a hassle when applying multiple suggestions to the same file.

Fortunately, Aider seems to be what I’ve been looking for: generating and editing source code by prompting while directly applying the changes in my local source code files (no more copy-pasting between tools).

Make Sure to Have Ollama Running

To use Aider effectively, ensure you have Ollama running. Ollama is an AI model that Aider utilizes for generating and modifying source code.

To start Ollama, use the following command in your terminal:

ollama start
o
Installing Aider

To install Aider, follow these steps:

    Install Python (if not already installed):

    bash

sudo apt-get install python3

Install Aider using pip:

bash

pip install aider

Verify the installation:

bash

    aider --version

Using Experimental Web UI

I prefer using CLI-based tools, which makes my terminal usage quite busy (with vim, git, build scripts, monitoring tools, etc.). Since I'll be using Aider only sporadically for now, I prefer having it elsewhere (alt+tabbing) to deal with one less process in my terminal workflow.

Although it's still experimental, the web UI works great.

To start the web UI, run:

bash

aider --webui

This will open Aider in your default web browser, providing a more user-friendly interface for generating and editing code.
Generating an Entire API with It

To generate an entire API with Aider, follow these steps:

    Start Aider:

    bash

aider

Prompt Aider for the API generation:

In the Aider interface, you can prompt it to generate an API. For example:

plaintext

    Generate a RESTful API in Python using Flask with endpoints for CRUD operations on a "User" resource.

    Review and edit the generated code:

    Aider will provide the generated code directly in your source files. Review and make any necessary adjustments.

    Run and test your API:

    Ensure your API is working correctly by running it and performing tests on the endpoints.

By following these steps, you can leverage Aider to streamline the process of generating and modifying source code directly within your files.
